# Void Archive

Научно-фантастическая инкрементальная/стратегическая игра на движке Godot. Проект сочетает в себе динамику менеджмента коррупции, пост-процессинг глитч-шейдеров, полноэкранную навигацию без вкладок и разветвленные древовидные графы прогрессии.

> 🛠️ **Статус проекта:** Игра находится в стадии активной разработки и проектирования. Полноценного, стабильного билда для скачивания пока нет — проект собирается, тестируется и полируется буквально на лету.

## 🌌 Основные механики

### 🔮 Производство и цикл Коррупции
Балансируйте между эффективностью и системным крахом. Инфраструктура генерирует **Вычисления**, однако высокая интенсивность производства ускоряет рост **Коррупции**.
* **Влияние:** Высокий уровень коррупции дает мощный бонус к глобальной выработке (`mult_production`), но пропорционально увеличивает риск возникновения критических аномалий.
* **Стабилизация:** Направляйте ресурс **Вычисления** на подавление и демпфирование кривой коррупции.
* **Тюнинг баланса (L3):** Накопление коррупции переведено на сглаженную логарифмическую кривую с асимптотой к $1.0$ (параметр `ACCRUAL_K = 0.00003`), что обеспечивает контролируемый темп игры на поздних этапах.

### 📡 Сигналы и Глитчи (Аномалии)
Вселенная реагирует на вашу активность. В игре динамически генерируются аномалии, шанс появления которых взвешен текущим уровнем коррупции.
* **Сигналы:** Положительные аномалии, дающие мгновенный приток данных (`instant_data_seconds`) или глобальные множители.
* **Gлитчи:** Деструктивные аномалии, дестабилизирующие производство. Одновременно может быть активна максимум одна аномалия.

### ⏳ Перезагрузка временной линии (Престиж)
Когда симуляция становится слишком нестабильной, запустите **Перезагрузку временной линии**, чтобы конвертировать текущий прогресс в **Хроно-эхо**.
* Тратьте Хроно-эхо в мета-дереве перков для перманентного улучшения систем.
* Доступные улучшения: автоматизация автокликеров, бусты глобальных множителей и механики задела (сохранение части ресурсов и прогресса при сбросе).

## 🎨 Графика и шейдеры

* **Пост-процесс ЭЛТ-шейдер:** Кастомный экранный шейдер, имитирующий аналоговый катодно-лучевой монитор.
* **Динамические глитчи:** Визуальные артефакты — дрожание, хроматическая аберрация, скан-линии и выпадения строк — динамически усиливаются по мере роста уровня Коррупции.

## 🖥️ Интерфейс и архитектура

### Единый компонент `TreeGraph`
Устаревшая табличная структура полностью заменена на производительный холст интерактивного графа.
* **Плавная навигация:** Поддержка перетаскивания и панорамирования холста (drag-pan) с помощью левой кнопки мыши (ЛКМ).
* **Кривые Безье:** Связи между узлами (`requires`) математически просчитываются и рендерятся в виде кубических кривых Безье, соответствующих цвету конкретной ветки.
* **Статусы узлов:** Наглядная визуализация состояний (*Изучено*, *Доступно*, *Заблокировано*), дополненная подробными тултипами с описанием эффектов и стоимости.

### Интерфейс
* **Адаптивный интерфейс:** Главное окно поддерживает динамическое изменение размера (минимальное разрешение `960×600`) с режимом растягивания `expand`.
* **Оптимизация рендера:** Тяжелые ноды `ColorRect` заменены на стандартные `Panel`, управляемые скругленными стилями `StyleBoxFlat`.

## 📂 Технологический стек и база данных

* **Движок:** Godot Engine (Control-ноды, кастомный отрисовщик CanvasItem).
* **Архитектура:** Полноэкранная навигация без вкладок, переключение экранов реализовано через менеджмент видимости (`_ops_screen` $\leftrightarrow$ `_tree_screen`).
* **Дата-ориентированная прогрессия (`ResearchDB`):**
  * Позиционирование узлов на холсте через координаты `pos` (Vector2).
  * Корневые заглушки (`stub: true`, узлы `c_root` / `v_root`), выступающие шлюзами для ветвей **Сознания** и **Пустоты** (Level 5), которые открываются при срабатывании триггера `void_detected`.
